Staging Area (INDEX)

El area de control de cambios, corresponde a un archivo que se encuentra en la carpeta .git y se utiliza para realizar las capturas de pantalla del codigo(snapshot) de los diferentes archivos del proyecto. Aqui se realiza el seguimiento de los mismos para realizar la confirmacion y poder acceder a los diferentes cambios dentro de la historia de nuestro proyecto. Es importante tener en cuenta que todo archivo que no sea agregado al INDEX, no se podr√° recuperar en caso de ser eliminado.

  • git statusMuestra el estado de los archivos con respecto a la ultima confirmacion, en caso que exista
    • untracked:al archivo no se le esta realizando seguimiento
    • staged:los archivos fueron agregados, pero aun no estan confirmados
    • modified:el archivo difiere con respecto a lo almacenado en la ultima confirmacion
    • commited:los cambios fueron almacenados
  • git add <fileagrega al INDEX el/los archivo/archivos seleccionado
  • git rm --cached <filequita del INDEX el archivo seleccionado
  • git restore --staged <fileremueve los cambios de los archivos agregados al INDEX

Repository

Area donde se almacenan los cambios, mediante confirmaciones, en un formato BLOB(Binary Large Object File). Solamente se confirman las modificaciones agregadas al INDEX; y mediante apuntadores podemos navegar dentro del historial.

  • git commitabre el editor para realizar confirmacion
  • git blame <filemuestra linea por linea los autores de las confirmaciones, en caso que existan
  • git logmuestra el historial de confirmaciones
